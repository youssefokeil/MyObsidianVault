# Project Management Processes
0. **Project Pre-initiation**
	1. determine scope, time and cost constraints of project
	2. Identify project sponsor
	3. select project manager
	4. develop business case for a project
	5. meet with project manager to review process and expectations
	6. determine if project should be divided into two or more smaller projects
1. **Initiating Processes**
	1. Holds a kick-off meeting, so that stakeholders can meet each other, review goals of the projects and discuss future plans.
	2. *Outputs:* Charters that are normally short and include key project information and stakeholder signatures.
2. **Planning Processes**
	1. guide execution
	2. *Outputs:*  [[Work Breakdown Structure (WBS)]], Project schedule in the form of [[Gantt Chart]][^2] and list of prioritized tasks.
3. **Executing Processes**
	1. Takes the most time and resources to perform project execution
	2. Project Managers Must use their leadership skills to handle the many challenges that occur during project execution
	3. *Outputs:* a milestone report[^3] can help focus on completing major milestones & many project sponsors focus on deliverables.
4. **Monitoring and Controlling Processes** 
	1. Measuring progress toward project objectives, deviation from the plan
	2. Affects all other process groups
	3. *Outputs:* performance reports, change requests & updates to various plans
5. **Closing Processes**
	1. Gaining stakeholder and customer acceptance of final products and services.
	2. Even if project is not completed, they should be closed to learn from the past
	3. *Outputs:* includes project archives and lessons learned, part of organizational process assets, (sometimes) a final report and presentation to sponsor/CEO
